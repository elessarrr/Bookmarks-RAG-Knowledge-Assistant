# Master Context Document
**Project**: Bookmarks RAG Knowledge Assistant  
**Date**: 2026-07-18  
**Purpose**: Architectural snapshot after PRD 0003 portfolio credibility hardening

---

## Current repository structure

- FastAPI app under `app/` with ingestion, DuckDB storage, retrieval, and Ollama generation
- React/Vite UI under `frontend/`; Docker multi-stage build copies `frontend/dist` into the API image
- Offline eval harness under `evals/` with independent RAGAS judge and chunking comparison
- Runtime defaults in `config.yaml`; docs truth surface in `README.md` and `ARCHITECTURE.md`

## Architectural changes in this pass

### Robots enforcement restored
- `app/ingestion/fetcher.py` parses and caches per-origin `robots.txt` via `RobotFileParser`
- Explicit Disallow blocks page GET and returns `FetchResult(error="Blocked by robots.txt")`
- Unreachable robots files fail open with a warning (personal-tool UX)

### Static asset serving
- `app/main.py` prefers `frontend/dist` when `index.html` exists; otherwise creates/serves `static/` placeholder
- `/health` and `/api/*` are registered before the catch-all static mount
- `run.py` binds `0.0.0.0:8000` without reload so Docker can expose the built UI

### Config / model defaults
- Default generator is `llama3.2:3b`
- Eval judge remains independent (`qwen2.5:32b` by default)
- Config failure paths (missing file, invalid YAML, missing keys) are covered by unit tests

### Streaming honesty
- Streaming helpers remain library-only
- HTTP `/api/query` and React chat stay non-streaming
- README Known Limitations and ARCHITECTURE query path document that choice

## Data / control flow (post-hardening)

1. Upload bookmark HTML → parse → robots check → fetch allowed pages → clean → chunk → embed → DuckDB
2. Query → embed → exact cosine search → prompt with retrieved sources → non-stream Ollama generate → JSON answer + sources
3. Docker: Vite build → `/app/frontend/dist` → FastAPI StaticFiles at `/`

## Configuration requirements

- Python ≥ 3.11 and dependencies from `requirements.txt` / `requirements-dev.txt`
- Ollama with at least `llama3.2:3b` for local demos; optional `gpt-oss:20b` override
- Eval judge still expects a distinct model family (`qwen2.5:32b` or documented alternate)

## Impact on existing functionality

- Ingestion may skip more URLs when sites explicitly disallow crawling
- Docker demos now show the React shell instead of the API placeholder
- Local laptop defaults are smaller/faster to pull; quality override remains documented
- Interviewers get an honest Known Limitations surface without architecture expansion
